# 29 LoRA 与 SFT 基础

这一模块开始补模型适配这条线里最常见的一组概念：

- `SFT`
- `LoRA`

前面你已经补了：

- Agent 工程
- RAG
- MCP / A2A
- Prompt 与 routing
- 部署

这一节要补的是：

- 什么情况下 prompt 和 RAG 还不够
- 什么是监督微调
- 什么是 LoRA
- 为什么很多团队不会一上来就全量微调

## 1. 为什么会出现微调

很多任务只靠：

- prompt
- tool use
- RAG

就已经能做得不错。

但有些场景还会继续遇到问题：

- 输出风格不稳定
- 某类任务长期表现不准
- 领域术语理解不够稳
- 结构化抽取错误率较高
- 固定格式任务一致性不够

这时就会考虑：

- 让模型通过训练更贴近目标任务

## 2. 什么是 SFT

SFT 可以先理解成：

- 用“输入 -> 理想输出”的样本继续训练模型

它的全称通常是：

- Supervised Fine-Tuning

也就是监督微调。

核心直觉很简单：

- 你给模型看一批高质量示例
- 让模型更倾向于学会这种回答方式

## 3. SFT 在学什么

SFT 不是让模型重新学一遍所有世界知识。

更常见的是学这些东西：

- 某类任务的回答格式
- 某类领域任务的处理方式
- 某些稳定输出习惯
- 某些特定指令跟随能力

例如：

- 医疗问答格式
- 金融报告摘要格式
- 指定 JSON 输出格式

## 4. 什么是训练样本

训练样本通常可以先理解成：

- 一条输入
- 对应一条理想输出

例如：

```text
输入：请总结公司本季度利润变化
输出：利润同比增长 18%，主要原因是...
```

很多时候，微调质量不取决于“样本多不多”，而更取决于：

- 样本是否干净
- 样本是否一致
- 样本是否真的代表目标任务

## 5. 什么是 LoRA

LoRA 可以先理解成：

- 一种更轻量的微调方式

它的核心思想不是把整个大模型都大改一遍，而是：

- 只训练一小部分附加参数

所以它通常比全量微调更省：

- 显存
- 训练成本
- 存储成本

## 6. 为什么 LoRA 常见

因为真实项目里，很多团队并不具备大规模全量微调条件。

他们更常见的目标是：

- 低成本适配一个领域任务

这时 LoRA 很有吸引力，因为：

- 训练成本更低
- 部署更轻
- 试错更快

## 7. 如何理解“全量微调”和“LoRA 微调”的区别

当前阶段先这样理解：

### 全量微调

- 改动整个模型参数
- 成本高
- 资源要求高

### LoRA 微调

- 只训练少量附加参数
- 更轻量
- 更适合多数业务团队

## 8. 什么场景可能需要 SFT / LoRA

常见场景：

- 某个固定任务反复做
- 输出格式必须稳定
- 领域术语和推理习惯需要长期固化
- prompt 调很久还是不稳定

例如：

- 医疗结构化抽取
- 法律问答模板输出
- 金融分析格式化总结

## 9. 什么场景不该急着微调

这点非常重要。

很多问题其实不该先怪模型。

常见更优先排查的层次是：

1. prompt 是否清楚
2. 工具调用链路是否正确
3. RAG 检索是否准确
4. 输出约束是否合理

如果这些基础都没做好，直接微调往往收益不大。

## 10. 微调和 RAG 是什么关系

这两个经常被混淆。

### RAG

- 解决的是知识来源问题
- 重点是“先查再答”

### 微调

- 解决的是行为模式和任务适配问题
- 重点是“让模型更会按某种方式做事”

一句话理解：

- RAG 更像给模型补资料
- 微调更像给模型改习惯

## 11. 微调和 prompt 又是什么关系

很多场景下，顺序通常应该是：

1. 先调 prompt
2. 再调工具和 RAG
3. 最后再考虑微调

因为：

- prompt 成本最低
- RAG 更适合补知识
- 微调成本更高，周期更长

## 12. 什么是指令数据集

你可以先理解成：

- 专门为“模型按指令完成任务”准备的数据集

它通常长得像：

- instruction
- input
- output

这类格式在 SFT 里非常常见。

## 13. 微调时最看重什么

当前阶段先抓住这几个：

1. 数据质量
2. 数据一致性
3. 任务定义清楚
4. 评测方式明确

如果数据本身混乱，微调很可能把问题放大。

## 14. 微调之后要看什么

不是训练完就结束。

还要看：

- 目标任务有没有更稳
- 泛化有没有变差
- 输出格式是否更一致
- 成本和收益是否值得

所以微调不是“做了就一定更强”，而是：

- 对特定任务更适配

## 15. 当前阶段最该先掌握什么

先抓住这几个：

1. SFT 是监督微调
2. LoRA 是轻量微调
3. 微调主要解决任务适配和行为稳定性
4. RAG 和微调解决的问题不同
5. 大多数业务问题不该一上来就做微调

## 16. 学完这一模块后你应该能做到

- 理解 SFT 和 LoRA 的基本定位
- 理解为什么 LoRA 在业务场景里常见
- 理解微调和 RAG、prompt 的区别
- 知道什么时候该优先微调，什么时候不该
- 为后续更深入的模型训练知识打基础
