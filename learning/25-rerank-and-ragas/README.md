# 25 Rerank 与 RAGAS

这一模块承接上一节的检索层。

上一节你已经知道：

- dense retrieval
- sparse retrieval
- hybrid search
- metadata filter

这一节继续补：

- 为什么“召回了”还不够
- 什么是 rerank
- 为什么要评测 RAG
- 什么是 `RAGAS`

## 1. 为什么检索完还不够

很多初学者会以为：

- 检索取到 top-k
- 直接塞给模型
- 事情就结束了

真实情况往往不是这样。

因为 top-k 里虽然“相关”，但相关性强弱并不一样。

常见情况：

- 有些 chunk 很相关
- 有些 chunk 只是沾边
- 有些 chunk 包含关键词，但对当前问题帮助不大

如果把这些结果不加区分地全塞给模型，会出现：

- 噪声变多
- 真正关键片段被淹没
- 回答质量下降

## 2. 什么是 rerank

rerank 可以先理解成：

- 对已经召回的候选结果再排一次序

也就是：

1. 第一轮先粗召回
2. 第二轮再精排

所以 rerank 不是替代检索，而是接在检索后面。

## 3. 为什么需要 rerank

因为第一轮检索通常更偏“快”和“召回率”。

它的目标是：

- 先尽量别漏掉可能相关的内容

但这不代表顺序已经最好。

rerank 的目标则更偏：

- 把真正最适合当前问题的结果排到前面

## 4. 一个最小 rerank 直觉

你可以先记成：

```text
用户问题
  -> retrieval top-k
  -> rerank
  -> 取前 n 条高质量结果
  -> 交给模型回答
```

## 5. rerank 和 top-k 的关系

两者不是一回事。

### top-k

- 是第一轮先取多少候选

### rerank

- 是对这些候选重新排序

例如：

- retrieval 先取 top-10
- rerank 后只保留前 3 条

这是很常见的做法。

## 6. 什么场景更需要 rerank

比较常见：

- 文档很多
- chunk 很多
- hybrid search 召回结果来源复杂
- 问题表达比较细
- 需要高精度回答

这类场景里，只靠第一轮召回往往不够稳。

## 7. 为什么 RAG 需要评测

因为 RAG 系统不是只看“模型回答像不像”。

你还要判断：

- 检索对不对
- 召回够不够
- 回答有没有基于检索内容
- 有没有胡编

如果没有评测，优化会变成拍脑袋。

## 8. RAG 评测通常在看什么

当前阶段先抓住几个核心方向：

- 检索质量
- 回答相关性
- 回答是否忠于参考资料
- 回答是否覆盖了问题要点

也就是：

- 不是只看“写得像不像”
- 还要看“是不是基于正确证据”

## 9. 什么是 RAGAS

`RAGAS` 可以先理解成：

- 专门用于评测 RAG 系统效果的一套方法 / 框架

它关注的不是单一指标，而是从多个角度看 RAG 质量。

你当前阶段不用先记住它全部实现细节。

先建立直觉：

- RAGAS 是帮助你系统化评估 RAG 的

## 10. 为什么只看最终回答不够

因为一个回答看起来不错，不代表链路真的健康。

例如：

- 检索结果其实不对，但模型靠常识“蒙对了”
- 回答语气流畅，但没有真正基于文档
- 回答覆盖了一部分问题，但遗漏关键点

如果只看主观感觉，很容易误判系统质量。

## 11. RAG 优化通常怎么做

一个更合理的顺序通常是：

1. 看召回结果
2. 看 rerank 后顺序
3. 看最终塞给模型的上下文
4. 看最终回答
5. 再用评测指标判断改动是否真的变好

这比单看最终答案更可靠。

## 12. Rerank 和 RAGAS 在整条链路中的位置

你可以把它们这样放：

### rerank

- 属于在线检索链路的一部分

### RAGAS

- 属于离线评测和优化链路的一部分

也就是：

- rerank 直接影响线上召回质量
- RAGAS 帮你判断系统到底有没有优化成功

## 13. 当前阶段最该先掌握什么

先抓住这几个：

1. retrieval 负责先召回
2. rerank 负责精排
3. 最终回答质量取决于检索上下文质量
4. RAG 系统需要独立评测
5. RAGAS 是常见的 RAG 评测框架思路

## 14. 学完这一模块后你应该能做到

- 理解为什么召回后还需要 rerank
- 能区分 retrieval 和 rerank 的职责
- 理解为什么 RAG 不能只凭主观感觉优化
- 理解 `RAGAS` 在评测链路中的位置
- 为后面更完整的 Agent/RAG 系统设计打基础
