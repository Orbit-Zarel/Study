# 05 Java 文件读写入门

文件读写是很多后端功能的基础，比如：

- 读取配置文件
- 写日志
- 导入导出数据
- 处理上传文件

你现在不用一开始就记住所有 API，先建立最核心的理解就够了。

## 1. 什么是 I/O

`I/O` 是 `Input / Output` 的缩写。

可以先这样理解：

- Input：把外部数据读进程序
- Output：把程序里的数据写到外部

例如：

- 从文件中读取文本
- 把结果写到一个文件里

## 2. 为什么文件读写重要

因为真实项目里，程序不可能永远只在内存里运行。

它总会和外部世界打交道，例如：

- 磁盘文件
- 网络
- 数据库
- 消息队列

文件是最容易入门的一种 I/O 场景。

## 3. Java 中最基础的文件操作

你现在先认识两个方向：

- 读取文件
- 写入文件

在较新的 Java 写法里，`Files` 工具类很常用。

例如：

```java
String content = Files.readString(path);
Files.writeString(path, "hello");
```

## 4. 为什么文件操作常和异常一起出现

因为文件相关操作很容易失败：

- 文件不存在
- 没有权限
- 路径写错
- 编码不对

所以你会发现，文件 I/O 和异常处理几乎总是一起出现。

## 5. 路径怎么理解

路径就是“文件所在的位置”。

例如：

- `learning/notes.txt`
- `C:/Orbit_Zarel/Study/learning/notes.txt`

相对路径：

- 相对于当前工作目录

绝对路径：

- 从磁盘根位置开始写完整地址

## 6. 学完这一模块后你应该能做到

- 读懂最基础的文件读写代码
- 知道如何创建一个文本文件并写入内容
- 知道如何把文件内容再读回来
