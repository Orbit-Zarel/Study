# 10 Redis 基础入门

这一模块的目标不是马上把 Redis 的所有数据结构都学完，而是先帮你建立非常关键的直觉：

- Redis 是什么
- 为什么后端项目里经常会用到 Redis
- 什么叫缓存
- 为什么“有数据库了还不够”

## 1. 什么是 Redis

Redis 可以先简单理解成：

- 一个非常快的键值存储系统
- 常被用来做缓存
- 也常被用来做计数器、限流、会话、排行榜等能力

你当前阶段先把它想成：

- “一个速度很快、常放在数据库前面的内存型存储”

## 2. 什么是 key-value

Redis 最经典的思路就是：

- 用一个 `key` 找到一个 `value`

例如：

```text
user:1:name -> alice
task:1001:status -> RUNNING
```

这里：

- `user:1:name` 是 key
- `alice` 是 value

你可以把它理解成一种“查字典”的感觉。

## 3. 什么是缓存

缓存可以先理解成：

- 把“常用、重复访问的数据”先放到更快的地方

这样下次再访问时，就不用每次都去查更慢的地方。

例如：

- 原始数据在数据库里
- 热点数据先放进 Redis
- 下次请求优先查 Redis

## 4. 为什么有数据库还要 Redis

因为数据库虽然重要，但它不是用来解决所有性能问题的。

如果每次请求都直接查数据库，可能会遇到：

- 延迟更高
- 数据库压力大
- 热点数据被反复查询

而 Redis 常用来解决：

- 热点查询加速
- 减少数据库压力
- 临时状态保存

## 5. 一个非常常见的缓存流程

你可以先记住这个最经典的流程：

1. 用户请求数据
2. 后端先查 Redis
3. 如果 Redis 里有，就直接返回
4. 如果 Redis 里没有，再查数据库
5. 查到数据库结果后，再把结果写回 Redis

这就是非常典型的“缓存旁路”思路。

## 6. Redis 为什么快

你当前阶段先抓最核心的直觉就够：

- Redis 主要把数据放在内存里
- 内存访问通常比磁盘更快
- 它的数据结构和实现也做了很多性能优化

所以它非常适合：

- 快速读写
- 高频访问场景

## 7. Redis 常见使用场景

### 缓存热点数据

例如：

- 用户信息
- 商品详情
- 任务状态

### 计数器

例如：

- 某接口今天被调用了多少次

### 限流

例如：

- 某用户 1 分钟内最多请求 10 次

### 临时状态

例如：

- 登录验证码
- 短期 session 信息

## 8. 缓存不是永远正确的，要注意一致性

这是一个很重要的工程意识。

如果数据库里的数据更新了，而 Redis 里还是旧值，就会出现：

- 缓存和数据库不一致

所以后面真正做项目时，你会经常考虑：

- 更新数据库后要不要删缓存
- 缓存多久过期
- 某些数据是否适合缓存

当前阶段先知道：

- 缓存可以提升性能
- 但也会带来一致性和失效策略问题

## 9. 先认识几个常见词

### 过期时间（TTL）

表示：

- 一个缓存值能存活多久

例如：

- 60 秒后自动失效

### 热点数据

表示：

- 被频繁访问的数据

### 缓存命中

表示：

- 请求的数据刚好在缓存里，直接拿到了

### 缓存未命中

表示：

- 缓存里没有，需要回源查数据库

## 10. Redis 和你后面主项目的关系

你后面的 AI 平台后端项目里，Redis 很可能会用于：

- 缓存热点任务查询
- 做用户级限流
- 保存某些短时状态
- 减轻数据库压力

所以这部分不是“额外知识”，而是和主项目直接相关。

## 11. 学完这一模块后你应该能做到

- 理解 Redis 是干什么的
- 理解缓存为什么存在
- 看懂最基础的“先查缓存，再查数据库”的思路
- 知道缓存会带来一致性问题
